Tensorflow Object Detection API 설치하기

  • 1편에서 설치한 google-cloud-sdk를 설치한 경로와 같은 곳에, TensorFlow의 Models를 Clone한다.

  • 그 뒤에는 models/research/object_detection/g3doc/으로 이동해서 installation.md 을 열어본다. 절대설명하기귀찮아서가가아니라👀 installation.md 를 보고, Dependencies, COCO API, Protobuf, Protobuf Compilation, PYTHONPATH 등록까지 순서대로 설치 및 실행한다.

  • 설치하는 과정 중에, 무언가가 tensorflow에는 없다는 에러가 뜬다면 설치되어 있는 tensorflow의 버전이 2.0 이상이라서 호환이 안되는 경우일 수 있다. 그런 경우 아래와 같이 tensorflow 버전을 낮춰주면 실행이 된다. 필자는 아래 예시처럼 1.14.0으로 버전을 낮춘 뒤 설치했다.

  • 잘 설치가 됐는지, 아래 명령어를 models/research로 이동해서 실행해보자.

    tensorflow가 최신이 아니라서 곧 지원을 중단할 것이라는 엄청난 Future warnings를 뒤로하고, 아래와 같은 결과를 볼 수 있다면 Tensorflow Object Detection API를 성공적으로 설치완료.👏👏

object_detection_tutorial.ipynb

  • models/research/objection_detection/ 폴더 안에 보면, object_detection_tutorial.ipynb 라는 파일을 발견할 수 있을 것이다. 이 파일을 jupyter notebook으로 실행시키면, Tensorflow Object Detection API를 사용해 볼 수 있다.

  • 이 파일을 열면, Setup의 첫부분에서 tensorflow의 버전을 최신 버전으로 설치해주자. 이미 최신 버전이라면 할 필요가 없다. 그 뒤의 과정은 쭉 생략하고 (위에서 installation.md를 통해 진행한 과정이기 때문에), Imports부분부터 시작해보자.

  • 원래 적혀 있는 코드 중, 아래와 같이 import pathlib만 추가해주자.

✔ Selecting a model

  • models/research/object_detection/g3doc 폴더에 가면, detection_model_zoo.md 라는 파일을 우리는 확인할 수 있다. 이 파일에서 우리는 COCO dataset로 pre-train된 모델들의 목록을 아래와 같이 확인할 수 있다.

  • ⭐표시가 있는 것들이 바로, TPU로 학습이 가능한 모델들이다. 각자가 학습하기 원하는 모델을 클릭하면, 해당 pre-trained model의 학습된 graph 및 config 파일을 다운받을 수 있다.

  • 다운 받은 그래프 파일과 config 파일을 각자가 학습하고자 하는 데이터에 맞게 어떻게 사용할 수 있는지는 이후Posting에서 살펴보도록 하겠다.

  • 필자는 속도는 조금 느리지만, 비슷한 mAP 성능 대비 빠른 속도를 내는, ssd_mobilenet_v1_fpn_coco를 선택했다.

  • 다음 포스팅에서는 각자가 가지고 있는 Annotation File을 어떻게 모델이 학습할 수 있는 파일인 tfrecord파일로 만들 수 있는지 살펴 보도록 하겠습니다! 😀
